# Ещё одна дорога домой, или как получить Российский IP

## []()Российский IP на телефоне без VPS и без VPN провайдера

### []()Зачем это нужно

[]()Задача возникла из простой жизненной ситуации - необходимости находясь в Европе вносить показания счетчиков, получать платежки, заходить на Российские сервисы которые внезапно перестали пускать тех кто не с ними.  
Фактически для этого есть два решения - ВПНнаоборот за неизвестные мне деньги и сервис vpnobratno.info - прекрасный некоммерческий проект, увы, сильно зависящий от филантропов и Роскомнадзора на местах.  
Большая часть дальнейшего текста написана Клодом (Claude) - спасибо Anthropic, им же решались и технические вопросы по дороге. Моим был только выбор пути.

[]()Классический VPN от провайдера — удобно, но есть проблемы:

- []()Провайдеры VPN блокируются или деградируют — сегодня работает, завтра нет
- []()Доверие к чужому серверу — кто знает что они логируют
- []()Зависимость от чужой инфраструктуры и чужих решений
- []()Цена — за хороший сервис надо платить

[]()Альтернатива — построить свой туннель. Но обычно это требует VPS с российским IP, а это снова чужой сервер и ежемесячная оплата. Если у вас есть и всё устраивает - дальнейшее чтение - только удовлетворение любопытства.

[]()Мы нашли другой путь: **бесплатный российский IP через Яндекс Serverless Container и Cloudflare Zero Trust**, без VPS, без постоянно работающего сервера.

[]()Особенность решения — туннель идёт только до Яндекса (это российская инфраструктура, М9 и подобные дата-центры). Это даёт независимость от мелких внутрироссийских блокировок и ограничений на уровне провайдеров.

---

### []()Архитектура

```
Телефон (Какая то страна/Европа)
    ↓ WARP агент
Cloudflare Zero Trust
    ↓ туннель ya-bridge
Яндекс Serverless Container (Москва, 185.206.167.x)
    ↓ выход в интернет
Российский IP
```

[]()Ключевой элемент — **Hostname Routes** в Cloudflare Zero Trust (бета-функция). Она позволяет направить трафик к конкретным доменам через туннель, и сайты видят IP Яндекс контейнера — российский адрес.

---

### []()Что используется

- []()**Яндекс Serverless Container** — бесплатный уровень, оплата только за вызовы
- []()**Cloudflare Zero Trust** — бесплатный план
- []()**cloudflared** — открытый клиент Cloudflare туннеля
- []()**GitHub Actions** — бесплатная сборка Docker образа
- []()**WARP** — бесплатный агент Cloudflare для телефона

[]()Стоимость: практически нулевая при умеренном использовании.

### []()Что делаем

#### []()1. Яндекс Cloud

[]()Создаём Serverless Container с минимальными настройками:

- []()RAM: 128 MB
- []()CPU: 5%
- []()Таймаут: 600 секунд
- []()Подготовленных экземпляров: 0 (запускается по требованию)

#### []()2. Docker образ

[]()Минимальный Dockerfile:

```
FROM alpine:latest
RUN apk add --no-cache curl && \
    curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64 \
    -o /usr/local/bin/cloudflared && \
    chmod +x /usr/local/bin/cloudflared
COPY start.sh /start.sh
RUN chmod +x /start.sh
ENTRYPOINT ["/bin/sh", "/start.sh"]
```

[]()[start.sh](http://start.sh/):

```
#!/bin/sh
while true; do
  echo "External IP: $(curl -s https://ipinfo.io/ip)"
  sleep 30
done &
cloudflared tunnel --no-autoupdate run
```

[]()Образ собирается через GitHub Actions и пушится в Яндекс Container Registry.
Curl реализует иллюзию жизни изнутри - заодно позволяет видеть это в логе - без него возможно отключение контейнера - но это как обычно неточно.

#### []()3. Cloudflare Zero Trust туннель

1. []()Создать туннель `ya-bridge` в Zero Trust Dashboard
2. []()Скопировать токен туннеля в переменные окружения контейнера как `TUNNEL_TOKEN`
3. []()Запустить контейнер — он сам подключится к Cloudflare

#### []()4. Настройка Gateway

[]()Без этого WARP не работает:

```
Gateway → Firewall Policies → DNS → Add policy
```

[]()Условие: Domain matches regex `.*`
Action: Allow

#### []()5. Hostname Routes (ключевой шаг)

```
Networks → Tunnels → ya-bridge → Hostname routes → Create
```

[]()Добавляем домены которые должны видеть российский IP:

- []()`*.ru`
- []()`*.рф`
- []()и другие по необходимости

[]()Это бета-функция — туннель выбирается автоматически на основе существующего подключения.
Уже нет - но не пугайтесь если опять да.

#### []()6. WARP на телефоне

[]()Установить Cloudflare WARP, сейчас это называется Cloufflare One, подключить к своей Zero Trust организации.

### []()Как пользоваться

[]()Контейнер запускается по требованию. Чтобы его разбудить — открыть прямой URL контейнера в браузере. Страница будет крутить колесико — это нормально, контейнер держит соединение открытым.

[]()Пока страница открыта — WARP на телефоне использует российский IP для доменов из Hostname Routes.

[]()На Android удобно использовать **HTTP Shortcuts** — создать ярлык на рабочем столе, нажать один раз для запуска. Запускать при включенном тоннеле! - иначе оборвет соединение при включении.

[]()Выключить — просто отключить WARP агент на телефоне.

---

### []()Важные особенности

[]()**Не все сайты доступны.** Некоторые сервисы (госпорталы, банки) блокируют IP дата-центров — пропускают только домашние и мобильные адреса. Это ограничение не решается данным методом. Это сильно неточно - взято из тырнетов - но имейте ввиду.

[]()**Сессии при перезапуске.** Если контейнер перезапустится — IP контейнера может смениться. Сервисы привязывающие сессию к IP (госуслуги, некоторые банки) попросят авторизоваться заново.

[]()**Не все домены.** Нужно вручную добавлять домены в Hostname Routes. Можно добавить *`.com`,*`.net` и другие зоны широко.

---

### []()Почему это работает

[]()Cloudflare Hostname Routes (бета) перенаправляет трафик к указанным доменам через существующий туннель. Трафик выходит с IP контейнера — российского сервера Яндекса. Это не egress policy (которая только на Enterprise), а маршрутизация через туннель на уровне hostname.

[]()Все ИИ-помощники говорили что это невозможно, не сразу причем, а после долгих почти успешных попыток. Только Chat GPT сказал нет сразу. Оказалось — возможно, просто функция
которая всё спасла спрятана в бета-разделе, за что спасибо CloudeFlare.
